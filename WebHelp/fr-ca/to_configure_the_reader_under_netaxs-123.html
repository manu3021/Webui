Configurer le lecteur sous NetAXS-123

Pour configurer les paramètres du lecteur :

  1. Sur le tableau de bord de MPC, cliquez sur l’onglet Créer et modifier. La page CRÉER ET MODIFIER s’affiche.
  2. Dans le panneau de gauche, sélectionnez le site, puis cliquez sur le périphérique désiré associé au site.

    3. Remarque :

    Avant de modifier les paramètres du périphérique, veillez à bien enregistrer le périphérique sur le nuage.

  3. Dans le panneau de gauche, cliquez sur l’icône correspondant au périphérique NetAXS-123 pour développer et afficher toutes les portes disponibles.
  4. Faites un double-clic sur la porte désirée pour développer et afficher tous les lecteurs associés. La page Paramètres apparaît.
  5. Saisissez les informations nécessaires dans les champs suivants :
    6. Champ Description
    Nom du lecteur Entrez un nom unique pour identifier le lecteur.
    Désactiver un lecteur ou une porte Dans la liste déroulante, sélectionnez les propriétés utilisées pour régler le lecteur ou la porte en mode activé ou désactivé.
    Carte et PIN requis Autorise uniquement l’accès en cas de lecture de carte valide et de saisie d’un code PIN autorisé sur le clavier de la porte. La lecture de la carte et la saisie du code PIN peuvent être effectuées dans n’importe quel ordre.
    Carte ou PIN requis Autorise l’accès en cas de lecture de carte valide ou de saisie d’un code PIN autorisé sur le clavier de la porte.
    PIN uniquement Autorise uniquement l’accès en cas de saisie d’un code PIN autorisé sur le clavier de la porte.
    Carte uniquement Autorise uniquement l’accès en cas de lecture de carte valide.
    Verrouiller un lecteur ou une porte Ignore toutes les lectures de carte (sauf avec une carte VIP), ignore les entrées de porte mais autorise les sorties.
    Contrainte Autorise à déclencher un événement d’alarme et, en cas de configuration correspondante, une impulsion sur un appareil de sortie en cas de contrainte, par exemple quand l’opérateur est forcé de donner accès contre son gré à une personne non autorisée. Une contrainte nécessite un code PIN et un numéro de carte pour être reconnue.
    Antiretour L’utilisateur active la fonctionnalité antiretour sur le lecteur, ce qui nécessite une carte valide pour l’entrée et la sortie. Le titulaire de la carte doit utiliser la carte avec la bonne séquence ENTRÉE/SORTIE, ce qui signifie qu’une carte utilisée sur un lecteur d’ENTRÉE devra ensuite être utilisée sur un lecteur de SORTIE, ou vice-versa (une carte utilisée sur un lecteur de SORTIE devant ensuite être utilisée sur un lecteur d’ENTRÉE). Si la séquence ENTRÉE/SORTIE de l’utilisateur est invalide, un événement d’infraction antiretour est généré pour le type d’antiretour choisi (strict ou modéré). L’accès de l’utilisateur est alors refusé (strict) ou autorisé (modéré). <> Strict - vérifie le statut ENTRÉE/SORTIE avant d’autoriser l’accès. Une deuxième utilisation de la carte sur le même type de lecteur (ENTRÉE/SORTIE) entraîne une infraction antiretour stricte, et l’accès est refusé à l’utilisateur.
    <> Modéré - vérifie le statut ENTRÉE/SORTIE avant d’autoriser l’accès. Une deuxième utilisation de la carte sur le même type de lecteur (ENTRÉE/SORTIE) entraîne une infraction antiretour modérée, mais l’accès est autorisé à l’utilisateur.
    <> Sortie - s’applique aux lecteurs à l’intérieur de la zone à contrôle antiretour. Les titulaires de carte utilisent ces lecteurs pour quitter la zone à contrôle antiretour.
    <> Entrée - s’applique aux lecteurs à l’extérieur de la zone à contrôle antiretour. Les titulaires de carte utilisent ces lecteurs pour entrer dans la zone à contrôle antiretour.
